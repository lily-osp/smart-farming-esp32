# Required Arduino Libraries for Smart Farming System

This file lists all the required libraries for the ESP32 Smart Farming System.
Install these libraries through the Arduino IDE Library Manager before uploading the code.

## Core Libraries

### 1. LiquidCrystal_I2C
- **Purpose**: Control LCD 1602 display via I2C communication
- **Installation**: Library Manager → Search "LiquidCrystal I2C" → Install by Frank de Brabander
- **Version**: 1.1.2 or later
- **Usage**: Provides easy control of LCD displays with I2C backpacks

### 2. DHT sensor library
- **Purpose**: Read temperature and humidity from DHT11/DHT22 sensors
- **Installation**: Library Manager → Search "DHT sensor library" → Install by Adafruit
- **Version**: 1.4.4 or later
- **Dependencies**: Adafruit Unified Sensor (automatically installed)
- **Usage**: Handles communication with DHT temperature/humidity sensors

### 3. Adafruit Unified Sensor
- **Purpose**: Unified sensor abstraction layer (dependency for DHT library)
- **Installation**: Library Manager → Search "Adafruit Unified Sensor" → Install by Adafruit
- **Version**: 1.1.9 or later
- **Usage**: Required by DHT sensor library

## WiFi and Network Libraries (Online Version Only)

### 4. WiFi (Built-in)
- **Purpose**: ESP32 WiFi connectivity
- **Installation**: Built-in with ESP32 core
- **Usage**: Handles WiFi connection and management

### 5. HTTPClient (Built-in)
- **Purpose**: HTTP client for web requests
- **Installation**: Built-in with ESP32 core
- **Usage**: Sends data to cloud services like ThingSpeak

### 6. WebServer (Built-in)
- **Purpose**: HTTP web server for local web interface
- **Installation**: Built-in with ESP32 core
- **Usage**: Creates web interface for remote monitoring and control

### 7. ArduinoOTA (Built-in)
- **Purpose**: Over-The-Air firmware updates
- **Installation**: Built-in with ESP32 core
- **Usage**: Enables remote firmware updates without physical connection

## JSON and Data Libraries (Online Version Only)

### 8. ArduinoJson
- **Purpose**: JSON parsing and generation for API communication
- **Installation**: Library Manager → Search "ArduinoJson" → Install by Benoit Blanchon
- **Version**: 6.21.3 or later
- **Usage**: Handles JSON data for web API and cloud communication

### 9. Adafruit IO Arduino
- **Purpose**: Adafruit IO cloud service integration
- **Installation**: Library Manager → Search "Adafruit IO Arduino" → Install by Adafruit
- **Version**: 5.0.0 or later
- **Dependencies**: Adafruit MQTT Library (automatically installed)
- **Usage**: Sends sensor data to Adafruit IO cloud platform

## Installation Instructions

### Method 1: Arduino IDE Library Manager (Recommended)
1. Open Arduino IDE
2. Go to Tools → Manage Libraries
3. Search for each library name listed above
4. Click "Install" for each library
5. Wait for installation to complete

### Method 2: Manual Installation
1. Download library ZIP files from GitHub repositories
2. Go to Sketch → Include Library → Add .ZIP Library
3. Select the downloaded ZIP files
4. Restart Arduino IDE

### Method 3: Command Line (PlatformIO)
If using PlatformIO instead of Arduino IDE, add to platformio.ini:

```ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
lib_deps = 
    frank-de-brabander/LiquidCrystal_I2C@^1.1.2
    adafruit/DHT sensor library@^1.4.4
    adafruit/Adafruit Unified Sensor@^1.1.9
    bblanchon/ArduinoJson@^6.21.3
    adafruit/Adafruit IO Arduino@^5.0.0
```

## Library Versions and Compatibility

### Minimum Required Versions:
- LiquidCrystal_I2C: 1.1.2
- DHT sensor library: 1.4.4
- Adafruit Unified Sensor: 1.1.9
- ArduinoJson: 6.21.3
- Adafruit IO Arduino: 5.0.0

### ESP32 Core Version:
- ESP32 Arduino Core: 2.0.0 or later
- Recommended: 2.0.11 or later

## Troubleshooting Library Issues

### Common Problems:

1. **Library not found error**
   - Ensure library is installed correctly
   - Check spelling in #include statements
   - Restart Arduino IDE after installation

2. **Compilation errors**
   - Update to latest library versions
   - Check ESP32 core version compatibility
   - Clear Arduino IDE cache (File → Preferences → Show verbose output)

3. **I2C communication issues**
   - Verify I2C connections (SDA/SCL pins)
   - Check LCD I2C address (usually 0x27 or 0x3F)
   - Ensure proper power supply

4. **DHT sensor errors**
   - Check sensor connections
   - Verify DHT sensor type (DHT11 vs DHT22)
   - Ensure proper pull-up resistor

5. **WiFi connection problems**
   - Verify WiFi credentials in config.h
   - Check WiFi signal strength
   - Ensure 2.4GHz network (ESP32 doesn't support 5GHz)
