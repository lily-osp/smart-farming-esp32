Project Overview

Key Features

Automated Irrigation

Intelligent watering based on soil moisture levels with configurable thresholds.

Light Level Monitoring

Optional LDR sensor for ambient light detection with configurable thresholds for day/night cycles.

IoT Connectivity

WiFi connectivity with dual cloud service integration (ThingSpeak & Adafruit IO).

Safety Mechanisms

Watchdog timer, emergency stop, and pump runtime protection.

Modular Design

Flexible display options and configurable sensor combinations.

Complete Configuration Reference

Serial Communication Settings
SERIAL_OUTPUT_ENABLED
#define SERIAL_OUTPUT_ENABLED true

Controls whether serial output is compiled into the program. When false, all Serial.print() calls are removed during compilation, reducing memory usage and improving performance.

true Enable serial output (default)
false Disable serial output for production
SERIAL_BAUD_RATE
#define SERIAL_BAUD_RATE 115200

Sets the serial communication speed. Higher values allow faster data transmission but may cause issues with some USB-to-serial converters.

115200 Recommended speed (default)
9600 Slower, more compatible
Display Configuration
DISPLAY_TYPE
#define DISPLAY_TYPE DISPLAY_LCD_1602

Selects the type of display to use. Different displays have different pin configurations and capabilities.

DISPLAY_LCD_1602 16x2 character LCD (default)
DISPLAY_LCD_2004 20x4 character LCD
DISPLAY_NONE No display (headless mode)
LCD_I2C_ADDRESS
#define LCD_I2C_ADDRESS 0x27

I2C address for LCD display. Most common addresses are 0x27 and 0x3F. Use I2C scanner to find the correct address.

0x27 Most common address (default)
0x3F Alternative common address
Control Configuration
CONTROL_TYPE
#define CONTROL_TYPE CONTROL_ROTARY_ENCODER

Selects the type of manual control interface. This affects which pins are used and how user input is processed.

CONTROL_ROTARY_ENCODER Rotary encoder with button (default)
CONTROL_POTENTIOMETER Analog potentiometer
CONTROL_NONE Automated only, no manual control
ROTARY_ENCODER_PINS
#define ROTARY_ENCODER_CLK_PIN 18
#define ROTARY_ENCODER_DT_PIN 19
#define ROTARY_ENCODER_SW_PIN 21

GPIO pins for rotary encoder connections. CLK and DT are for rotation detection, SW is for button press.

CLK: Clock pin (default: GPIO 18)
DT: Data pin (default: GPIO 19)
SW: Switch pin (default: GPIO 21)
Sensor Configuration
DHT_SENSOR_TYPE
#define DHT_SENSOR_TYPE DHT22

Type of DHT temperature and humidity sensor. DHT22 is more accurate but more expensive than DHT11.

DHT22 High accuracy (±0.5°C, ±2% RH)
DHT11 Lower cost (±2°C, ±5% RH)
DHT_PIN
#define DHT_PIN 4

GPIO pin connected to DHT sensor data line. Must be a digital pin that supports digital input/output.

Default: GPIO 4
Avoid: GPIO 0, 1, 2, 3 (boot pins)
SOIL_MOISTURE_PIN
#define SOIL_MOISTURE_PIN 34

Analog pin for soil moisture sensor. ESP32 has multiple ADC pins available for analog readings.

Default: GPIO 34 (ADC1_CH6)
Alternative: GPIO 35, 36, 39
LDR_PIN
#define LDR_PIN 39

Analog pin for Light Dependent Resistor (LDR) sensor. Requires 10kΩ pull-down resistor for proper operation.

Default: GPIO 39 (ADC1_CH3)
Required: 10kΩ pull-down resistor
Actuator Configuration
PUMP_PIN
#define PUMP_PIN 2

GPIO pin for water pump control. Uses relay or MOSFET for switching. Pin 2 has built-in LED for visual feedback.

Default: GPIO 2 (built-in LED)
Requires: Relay or MOSFET module
LED_PINS
#define LED_RED_PIN 5
#define LED_GREEN_PIN 17
#define LED_BLUE_PIN 16

GPIO pins for status LED indicators. RGB LED or separate LEDs for different status indications.

Red: GPIO 5 (error/warning)
Green: GPIO 17 (normal operation)
Blue: GPIO 16 (system status)
Threshold Configuration
SOIL_MOISTURE_THRESHOLD
#define SOIL_MOISTURE_THRESHOLD 30

Soil moisture percentage below which irrigation is triggered. Lower values mean drier soil triggers irrigation.

Range: 0-100 (percentage)
Default: 30% (moderate dryness)
TEMPERATURE_THRESHOLD
#define TEMPERATURE_THRESHOLD 35

Temperature threshold in Celsius for heat stress detection. Above this temperature, additional irrigation may be triggered.

Range: -40 to 80°C
Default: 35°C (hot weather)
HUMIDITY_THRESHOLD
#define HUMIDITY_THRESHOLD 20

Humidity percentage below which dry air conditions are detected. Low humidity may trigger additional irrigation.

Range: 0-100 (percentage)
Default: 20% (very dry air)
LIGHT_THRESHOLD
#define LIGHT_THRESHOLD 500

Light intensity threshold for day/night detection. Below this value, system operates in night mode.

Range: 0-4095 (12-bit ADC)
Default: 500 (dim light)
IoT Services Configuration
IOT_SERVICES_ENABLED
#define IOT_SERVICES_ENABLED true

Master switch for all IoT services (ThingSpeak, Adafruit IO). When disabled, only the built-in web UI is available.

true Enable IoT services (default)
false Disable all IoT services
WEB_UI_ONLY_MODE
#define WEB_UI_ONLY_MODE false

Convenience setting to disable all IoT services and use only the built-in web interface. Automatically sets IOT_SERVICES_ENABLED to false.

false Normal mode with IoT services (default)
true Web UI only mode
THINGSPEAK_ENABLED
#define THINGSPEAK_ENABLED true

Enable/disable ThingSpeak cloud service for data logging and visualization.

true Enable ThingSpeak (default)
false Disable ThingSpeak
Requires: ThingSpeak API key and channel ID
Features: Data logging, charts, alerts
ADAFRUIT_IO_ENABLED
#define ADAFRUIT_IO_ENABLED true

Enable/disable Adafruit IO cloud service for IoT data management and dashboards.

true Enable Adafruit IO (default)
false Disable Adafruit IO
Requires: Adafruit IO username and key
Features: Real-time dashboards, feeds
Timing Configuration
IRRIGATION_DURATION
#define IRRIGATION_DURATION 5000

Duration in milliseconds for irrigation cycles. Longer duration provides more water but may cause overwatering.

Range: 1000-60000 ms
Default: 5000 ms (5 seconds)
SENSOR_READ_INTERVAL
#define SENSOR_READ_INTERVAL 1000

Interval in milliseconds between sensor readings. Shorter intervals provide more responsive control but increase power consumption.

Range: 500-10000 ms
Default: 1000 ms (1 second)
DISPLAY_UPDATE_INTERVAL
#define DISPLAY_UPDATE_INTERVAL 2000

Interval in milliseconds for display updates. Longer intervals reduce flickering but make display less responsive.

Range: 1000-10000 ms
Default: 2000 ms (2 seconds)
WATCHDOG_TIMEOUT
#define WATCHDOG_TIMEOUT 30

Watchdog timeout in seconds. System will restart if main loop doesn't respond within this time.

Range: 10-300 seconds
Default: 30 seconds

Setup Guide

1

Hardware Testing

Start by running the hardware test code to verify all connections and components.

TestCode/hardware/hardware_test.ino
2

Communication Testing

Test WiFi and cloud connectivity (for online version only).

TestCode/communication/communication.ino
3

Install Libraries

Install all required libraries via Arduino IDE Library Manager.

See libraries.txt for complete list
4

Open Project in Arduino IDE

Open the appropriate sketch file for your chosen version.

MainCode/offline/offline.ino # For offline version
MainCode/online/online.ino # For online version
TestCode/hardware/hardware_test.ino # For hardware testing
TestCode/communication/communication.ino # For communication testing
5

Configure System

Edit the configuration file to match your hardware setup and requirements.

MainCode/offline/config.h # For offline version
MainCode/online/config.h # For online version

API Documentation

REST API Endpoints

GET /api/status

Get current system status and sensor readings

{
  "systemOK": true,
  "soilMoisture": 45,
  "temperature": 25.3,
  "humidity": 60.2,
  "pumpActive": false,
  "uptime": 3600
}
POST /api/pump

Control pump manually

{
  "action": "start" | "stop",
  "duration": 30
}
GET /api/config

Get current system configuration

POST /api/config

Update system configuration

Troubleshooting & Support

Common Issues

System Not Starting

  • • Check power supply (minimum 2A)
  • • Verify all connections
  • • Run hardware test first

Sensors Not Reading

  • • Check wiring connections
  • • Verify power supply voltage
  • • Check sensor configuration
  • • LDR: Verify 10kΩ pull-down resistor
  • • LDR: Check GPIO 39 connection

WiFi Not Connecting

  • • Check WiFi credentials
  • • Verify signal strength
  • • Ensure 2.4GHz network
